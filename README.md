# Company-ERD-Diagram

COMPANY DATABASE ENTITY-RELATIONSHIP (ER) DIAGRAM DESIGN TASK
This task asks you to design an Entity-Relationship (ER) Diagram for a company database system that stores and organizes information related to employees, departments, projects, managers, supervisors, and dependents within a large company. The purpose of the ER diagram is to analyze the business requirements and represent the structure of the database visually by identifying the entities, their attributes, the relationships between them, and the constraints that control how the data is connected. The system must manage employee information such as Social Security Number (SSN), first name, last name, birth date, and gender, while also tracking the department in which each employee works, the projects they participate in, the number of working hours spent on each project, and the supervisor responsible for managing them. The database must also store department information including department number, department name, and department locations, while ensuring that every department has exactly one employee assigned as its manager together with the manager’s hiring date. In addition, the system should manage project details such as project number, project name, location, and city, while maintaining the relationship between projects and departments because every project must belong to one department. The task also includes handling dependent information for employees, where each dependent has a name, gender, and birth date, with the condition that dependent records exist only while the employee remains in the company, making the dependent entity a weak entity. Furthermore, the ER diagram must correctly represent all relationship cardinalities such as one-to-many and many-to-many relationships, participation constraints, recursive relationships for employee supervision, primary keys, foreign keys, and relationship attributes. The final ER diagram should provide a complete conceptual design of the company database that can later be transformed into relational tables and implemented using SQL in a database management system
